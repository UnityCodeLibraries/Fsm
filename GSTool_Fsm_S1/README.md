# T1
### 应用场景: 
- 当你需要一个状态转换系统(例如玩家, 怪物等的自动状态管理系统)时, 使用此Fsm, 该系统提供成品状态机以及可供改写的基本结构
### 如何使用: 
- #### 使用现有状态机: 4状态{Idle, Walk, Attack, Hurt}
1. 新建一个Gameobject并挂载Fsm
1. 使用测试脚本TestFsm
1. 使用WASD进入移动状态, J攻击, Y受伤
1. 你可以在组件面板看到状态与按键变化
- #### 新增状态: 
1. 新建一个状态并实现IState接口
1. 在Fsm注册这个状态
    - Fsm开始中加入此状态并设置优先级
    - 在更新条件更新状态判定条件
1. 现在你可以根据触发条件自动转换此状态
### 优缺点: 
- 优点: 
    - 现在还看不出来, 只有用于状态转换的优先级判定比较有利
- 缺点: 
    - 现在还做不了什么东西, 只是空的状态
